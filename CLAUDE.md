# Dotfiles Repository Guide

## Architecture

This repository uses **Nix Home Manager** for declarative management of packages, dotfiles, and git config.

- `flake.nix` — Nix flake inputs (nixpkgs, home-manager)
- `home.nix` — Main config: packages, `home.file` symlinks, `programs.git`
- `bootstrap.sh` — One-time setup: Nix install + `home-manager switch`

### How it works

1. Config files live under `config/` in the repo
2. `home.nix` declares `home.file` entries that map them to `~/` paths
3. `home-manager switch` copies files to `/nix/store/` and creates symlinks
4. Changes require running `home-manager switch --flake .#default --impure` to take effect

### Directory layout

| Directory | Purpose |
|-----------|---------|
| `config/bash/` | bashrc, profile, bash_alias, bash_profile |
| `config/bash/bashrc.d/` | Drop-in shell scripts (10-step numbering) |
| `config/nvim/` | Neovim config (lazy.nvim) |
| `config/tmux.conf` | tmux config |
| `config/claude/` | Claude Code settings template |
| `bin/` | Helper scripts (installed to `~/.local/bin/` via Home Manager) |

## Conventions

### Adding packages

Add to `home.packages` in `home.nix`:

```nix
home.packages = with pkgs; [
  ghq
  new-tool  # add here
];
```

### Adding config files

1. Place the file under `config/`
2. Add a `home.file` entry in `home.nix`:

```nix
home.file.".config/foo".source = ./config/foo;
```

### Adding bin scripts

Place executable scripts in `bin/`. They are automatically installed to `~/.local/bin/` via:

```nix
home.file.".local/bin" = { source = ./bin; recursive = true; executable = true; };
```

### bashrc.d/ drop-in scripts

- 10-step numbering: `10-fzf.sh`, `20-direnv.sh`, ..., `90-zoxide.sh`
- Loaded by `config/bash/bashrc` from `~/.config/bash/bashrc.d/`
- Machine-local scripts go in `~/.bashrc.d/` (not tracked)

### Git config split

- `~/.config/git/config` — Common settings, generated by Home Manager (`programs.git.settings` in `home.nix`)
- `~/.gitconfig` — Machine-specific settings (user, email, credentials). Not tracked.
- `~/.gitconfig-<host>` — Per-host overrides loaded via `includeIf`
- Run `git-setup-user` to configure machine-specific git identity and credentials

### Applying changes

After modifying any config file or `home.nix`:

```shell
home-manager switch --flake .#default --impure
```

## Review checklist

When modifying this repository, verify:

- [ ] New bashrc.d/ scripts have appropriate 10-step number prefix for load order
- [ ] No secrets committed (AWS keys, tokens, passwords). Secrets belong in `~/.gitconfig`, `~/.bashrc.d/`, or environment variables
- [ ] New packages are added to `home.packages` in `home.nix`
- [ ] New config files have corresponding `home.file` entries in `home.nix`
- [ ] README.md reflects the current tool list and setup instructions
- [ ] Quoting is correct in shell scripts (glob expansion, paths with spaces)
